## ✔ 목표

우선, GitHub Pages(Jekyll 기반 블로그)는 "접근 권한(비공개)" 기능이 없다. 즉, GitHub Pages로 배포되는 사이트는 항상 공개된다.
따라서, 나만의 목적(다른 사람은 못 보고, 나만 보고 싶은 글)에 따라 글의 공개를 우회하는 방법을 정리한다.

## ✔ 방법
우회하는 방법은 3가지로 볼수 있는데, 가장 활용성이 높은 _drafts 폴더를 이용하는 방법을 정의 하겠다.
| 1. _drafts 폴더 활용
| 2. Front Matter로 날짜를 미래로 설정 (사람이 못 보도록 숨김)
| 3. GitHub Repository를 Private로 바꾸기

## ✔ 특징
- GitHub Pages에 절대 배포되지 않음
- 로컬에서만 확인 가능
- 폴더 이름만 만들면 바로 사용 가능

1. 사용법
    - 루트에 _drafts 폴더 생성:

    ```markdown
    _root/
        └── _drafts/
           └── 2025-01-23-내-비공개-메모.md
    ```

2. 파일 내용은 일반 Jekyll 포스트와 동일

    ```markdown
    ---
    title: "비공개 테스트 문서"
    categories: [life]
    tags: [private]
    ---

    이 문서는 공개되지 않습니다.

    ```

3. 로컬에서 보기

    ```bash
    bundle exec jekyll serve --drafts
    ```

## ✔ 장점

- Github Pages에 절대 올라가지 않음
- 로컬에서는 실제 게시글처럼 렌더링됨
- 포스트 이동만으로 공개/비공개 전환 가능
- 공개하고 싶을 때 → _posts 폴더로 이동만 하면 됨


## 📌 Draft 글은 어디에 표시되나?

- Home 화면 / 카테고리 / 태그 페이지 등 모든 페이지에서 일반 글처럼 표시된다.
