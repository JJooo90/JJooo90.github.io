## Chirpy v7.x에서 접기(토글) 기능 사용하는 방법

블로그에서 글이 길어지다 보니, 특정 내용을 접어서 보여주는 “folding(접기)” 기능이 필요했다.  
그래서 Chirpy 테마에서 예전에 제공되던 `spoiler` 태그를 사용해보려 했는데, 예상과 다르게 빌드 오류가 발생했다.

---

## Chirpy의 `{% raw %}{% spoiler %}{% endraw %}` 태그는 더 이상 존재하지 않음

Chirpy 테마의 spoiler 태그는 **v6.x까지 제공되던 커스텀 태그였지만, v7.x에서 완전히 제거되었다.**

삭제된 이유는 다음과 같다.

- GitHub Pages 호환성 문제  
- Liquid 필터 처리 안정성 문제  

따라서 **현재 Chirpy(v7.x 이상)에서는 spoiler 태그를 사용할 수 없다.**

---

## 그렇다면 현재 Chirpy에서 접기 기능은 어떻게 구현할까?

정답은 **HTML의 `<details>` 태그를 직접 사용하는 것**이다.

---

## ✔ 기본 형태

```html
<details>
<summary>제목(클릭하면 펼쳐짐)</summary>

내용을 여기에 적는다.

</details>
```

---

## ✔ Markdown과 함께 쓰고 싶다면

`div markdown="1"` 은 HTML 내부에서도 Markdown을 정상적으로 렌더링하도록 Kramdown(Jekyll 파서)에게 지시하는 기능이다.

```html
<details>
<summary>설명</summary>

<div markdown="1">

여기에는 **Markdown** 문법을 자유롭게 사용할 수 있다.

    ```bash
    echo "hello"
    ```

</div>
</details>
```

## 📓 삽질 기록

<details>
<summary>열어서 보기</summary>

<div markdown="1">

### 🚧삽질 기록 1. spoiler 롤백 후 발생한 Sass 경고 발생

spoiler를 위해 Gemfile에서 다음 항목을 수정했다.
- gem "jekyll-theme-chirpy", "~> 7.4", ">= 7.4.1" 부분 추가 
- _config.yml에 plugins:jekyll-include-cache 만 추가

<!-- ![_config.yml 및 Gemfile 수정 내용](/assets/img/posts/blog/2025-12-05/img_01.png) -->
![_config.yml 및 Gemfile 수정 내용]({{ "/assets/img/posts/blog/2025-12-05/img_01.png" | relative_url }})


수정 후, 테스트 과정에서 아래와 같은 Sass 경고가 발생했다.

```
Deprecation Warning [import]: Sass @import rules are deprecated and will be removed in Dart Sass 3.0.0.
```

이 경고는 **Chirpy 테마 내부에서 @import가 사용되고 있기 때문에 발생하는 표준 Sass 경고**이며,  
사용자가 제거할 수 있는 성질의 것이 아니다.

VSCode에서 일부 경고를 숨기거나 출력 결과를 깔끔하게 만들 수는 있지만,  
**Jekyll 빌드 자체에서 이 메시지를 완전히 제거하는 것은 불가능하다.**

---

#### ✔ 경고를 줄이기 위한 환경 결론

##### 1) Gemfile에서 Sass Converter 버전 명시

```ruby
gem "jekyll-sass-converter", "~> 3.0"
```

##### 2) 캐시 정리 및 빌드 재실행

```bash
bundle exec jekyll clean
bundle exec jekyll serve
```

---

### 🚧삽질 기록 2. `_sass` 폴더 생성 후 `ignore.scss` 안에 `$quiet: true !default;` 작성  
→ 의미 없음 (Chirpy는 자체 Sass 빌드를 사용)

### 🚧삽질 기록 3. VSCode Settings.json에서 Sass 관련 경고 숨기기 시도

```bash
// Sass 관련 경고 비표시
"sass.compiler.quietDeps": true,
"sass.quietDeps": true,
"sass.suppressDeprecationWarnings": true,

// SCSS IntelliSense 경고 비활성화
"sass.lint.includeWarnings": false,
"scss.lint.deprecation": "ignore",
"scss.lint.import": "ignore"
```

### 🚧삽질 기록 4. Gemfile에 `jekyll-sass-converter 2.x` 설치 시도  
→ **스타일이 완전히 깨짐**  
→ Chirpy v7.x는 dart-sass 기반의 3.x 버전과만 호환됨


</div>
</details>

## 정리

- `spoiler` 태그는 Chirpy v7.x에서 완전히 삭제되었다.  
- 접기 기능은 **HTML `<details>` 태그**로 구현해야 한다.  
- (삽질 기록 1) Sass Deprecation Warning은 **숨길 수는 있지만 제거는 불가능**하다. 
- (삽질 기록 2) Chirpy와 Sass 버전을 변경하면 예상치 못한 스타일 파손이 발생할 수 있어 주의해야 한다.
