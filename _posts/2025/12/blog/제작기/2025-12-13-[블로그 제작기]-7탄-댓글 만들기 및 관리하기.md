## 이 글에서 해결하고 싶은 문제

GitHub Pages 블로그를 운영하면서 다음과 같은 요구가 생겼다.

- 모든 게시글에 **기본적으로 댓글 기능을 달고 싶다**
- 댓글이 달리면
    - 어떤 글에 달린 댓글인지 바로 알고 싶다
    - 한 페이지에서 댓글을 모아서 확인하고 싶다
- 댓글을 삭제하거나, 잠그거나, 관리자로서 통제할 수 있으면 좋겠다

이 글은 위 요구를 **추가 서버 없이**,  
**GitHub 블로그 환경에서 가장 안정적으로 해결하는 방법**을 정리한 것이다.

---

## GitHub 블로그에서 댓글이 어려운 이유

GitHub Pages 기반 블로그는 다음 제약이 있다.

- 서버가 없다
- 로그인 시스템이 없다
- DB를 직접 둘 수 없다

따라서 일반적인 블로그 댓글 시스템을 그대로 쓰기 어렵다.

이 환경에서 현실적인 해법은 **댓글을 “외부 시스템에 위임”하는 것**이다.

---

## 선택한 해법 요약

> **Giscus + GitHub Discussions**

이 조합은 다음 구조를 가진다.

- 블로그 화면의 댓글 UI → **Giscus**
- 실제 댓글 데이터 저장 → **GitHub Discussions**
- 댓글 관리 페이지 → **GitHub 저장소의 Discussions 탭**

즉,

> 블로그 댓글 = GitHub Discussions의 글과 댓글

이라는 구조다.

---

## 댓글 관리 페이지는 어디인가?

댓글을 관리하는 페이지는 **블로그가 아니라 GitHub 저장소**에 있다.

    GitHub 저장소 → Discussions 탭

이 페이지가 아래 3가지 역할을 모두 담당한다.

1. 댓글 관리자 페이지
2. 댓글 알림 확인 페이지
3. 어떤 게시글에 댓글이 달렸는지 보는 목록



---

## 댓글이 달리면 실제로 어떻게 보이나?

1. 누군가 블로그 게시글에 댓글을 단다
2. GitHub Discussions에:
    - 해당 게시글과 연결된 Discussion이 자동 생성되거나
    - 기존 Discussion에 댓글이 추가된다
3. Discussions 목록에서:
    - 제목 또는 경로를 통해 **어떤 게시글의 댓글인지 바로 확인 가능**

이로 인해:

- “댓글이 달린 게시글 목록”
- “최근 댓글이 달린 글”

을 한눈에 파악할 수 있다.

---

## 댓글을 어떻게 관리할 수 있는가?

GitHub Discussions 기반이기 때문에 관리자 권한으로 다음이 가능하다.

- 댓글 삭제
- 댓글 수정
- Discussion 잠금 (추가 댓글 차단)
- 답변 채택
- Reaction 관리
- 사용자 차단 (GitHub 기준)

별도의 관리자 UI를 만들 필요가 없다.

---

## 모든 게시글에 댓글을 기본으로 다는 방법

Chirpy 테마 기준으로:

    comments:
        active: giscus

이 설정이 있으면:

- 모든 게시글에 댓글이 **기본 활성화**
- 특정 글만 끄고 싶을 때는 Front Matter에:

    comments: false

를 추가하면 된다.

---

## 어떤 게시글의 댓글인지 헷갈리지 않는 이유

Giscus에서 다음 설정을 사용한다.

    mapping: pathname

이 설정의 의미는:

- 게시글 URL 경로 기준으로
- 댓글과 게시글을 1:1로 연결

그래서:

- 글 제목이 바뀌어도
- 파일명이 바뀌어도

**URL만 유지되면 댓글은 그대로 유지된다.**

---

## 실제 운영 흐름 (관리자 관점)

1. 게시글 작성 및 배포
2. 댓글이 달림
3. GitHub 알림 또는 Discussions 확인
4. Discussions 제목을 보고 게시글 식별
5. 댓글 답변 또는 관리
6. 필요하면 Discussion 잠금 또는 후속 글 작성

이 흐름으로  
댓글을 “부담 없이, 한 곳에서” 관리할 수 있다.

---

## 이 방식이 잘 맞는 블로그 유형

- 기술 블로그
- 개발 기록 위주 블로그
- GitHub Pages 기반 블로그
- 댓글 품질을 어느 정도 통제하고 싶은 경우

반대로:

- 실시간 대화
- 감성 댓글 위주

를 기대한다면 맞지 않을 수 있다.

---

## 결론

> GitHub 블로그에서 모든 글에 댓글을 달고, 그 댓글을 한 곳에서 관리하고 싶다면  **Giscus + GitHub Discussions 조합이 가장 현실적인 해답이다.**

이 구조를 한 번 만들어 두면, 서버 없이도 **댓글 관리가 가능한 블로그 운영**이 가능해진다.
