## 문제 정의

블로그 구조 개편을 진행하면서 카테고리 기준과 디렉터리 구조는 일정 수준 정리되었다.  
그러나 구조 정리 이후, **시리즈 형태로 작성된 게시글의 정렬 문제가 명확하게 드러났다.**

확인된 문제는 다음과 같다.

- 시리즈 게시글이 `date desc` 기준으로 정렬되어 10탄 → 9탄 → … → 1탄 순서로 노출됨
- 동일한 시리즈로 인식하고 있던 게시글 일부가 카테고리 페이지에서 누락되거나 의도하지 않은 위치에 노출됨

이 문제는 단순 UI 문제가 아니라, **연속된 시리즈 콘텐츠의 읽기 흐름을 훼손한다는 점에서 구조적 문제**였다.

다만, 다음 전제는 유지하고자 했다.
- `date` 필드는 실제 작성 시점을 의미하며, 정렬을 위해 인위적으로 수정하지 않는다.

> 즉, **기록의 의미를 유지하면서 시리즈 흐름만 제어할 방법**이 필요했다.

---

## 기대했던 이상적인 구조

초기에는 다음과 같은 구조를 기대했다.
- 카테고리는 콘텐츠의 큰 분류 기준
- 그 하위에서 시리즈 단위로 게시글을 묶어
- 사용자가 포스트를 읽는 도중 관련 글을 자연스럽게 이어서 탐색

> 즉, **카테고리 → 시리즈 → 개별 포스트**라는 흐름이 자연스럽게 드러나기를 기대했다.

---

## 시도한 접근 방식

문제 해결을 위해 다음과 같은 접근을 순차적으로 검토했다.
1. 카테고리 구조 정리  
2. Chirpy 기본 기능(`series` + `date`) 활용
3. 별도 Index 문서 도입
4. `series_order` 기반 정렬 도입

---

## 구조적 제약의 확인
### 1. Chirpy 카테고리 구조의 제약

Chirpy에서 카테고리 페이지는 테마 레벨(`_layouts/category.html`)에서 관리되며, 기본 정렬 기준은 `date`로 고정되어 있다.

> 즉, 사용자 정의 필드(`series_order`)를 카테고리 페이지의 정렬 기준으로 활용하는 것은 **테마 수정 없이는 구조적으로 불가능**
> 이로 인해 카테고리는 **시리즈 흐름을 표현하는 영역으로 적합하지 않다**는 결론에 도달했다.

---

### 2. series + date 조합의 한계

`series` 기능을 활용하면 포스트 간의 연관성은 표현할 수 있으나, 정렬 기준은 여전히 `date`에 의존한다.
따라서, 오름차순 정렬을 위해 `date`를 수정해야 하며, 이는 기록의 `무결성`을 해치는 선택이라고 판단했다.

---

### 3. Index 문서 도입의 비용

Index 문서를 사용자용 목차 페이지로 활용하는 방안도 검토했다.

그러나 이 방식은:
- 별도의 URL 진입을 요구하고
- 모든 포스트에서 Index 연계를 관리해야 하며
- HTML 기반 커스텀 구현이 필요해 유지보수 비용이 높아진다

현재 단계의 요구사항 대비 과도한 구조라고 판단했다.

---

## series_order 도입과 역할 분리

최종적으로 선택한 방식은 `series_order` 필드를 도입하는 것이다.
- `date`는 그대로 유지
- 시리즈 내부의 순서만 명시적으로 정의
- 포스트 상단 시리즈 목록은 `series_order` 기준으로 정렬

이를 통해 다음과 같은 **역할 분리**가 가능해졌다.
- 시리즈 내부 흐름 → 사람이 정의
- 카테고리 페이지 → 전체 게시글 타임라인
- 정렬 기준의 책임을 명확히 분리

> 결과적으로, 하나의 정렬 기준으로 모든 문제를 해결하려는 접근을 포기하고, 영역별로 책임을 분리하는 설계를 선택했다.

---

## Index 문서의 역할 재정의

Index 문서는 사용자 진입점이 아닌, **시리즈 운영 및 정렬 정책을 명문화하는 문서**로 정의했다.
- 시리즈 관리 기준
- `series_order` 사용 규칙
- 향후 확장 시 참고할 설계 의도

> 즉, Index 문서는 **Single Source of Truth 역할의 정책 문서**로 활용한다.

---

## 최종 구조 요약

| 영역 | 역할 | 기준 |
|---|---|---|
| 포스트 상단 시리즈 목록 | 시리즈 흐름 표현 | series_order |
| Index 문서 | 시리즈 운영 정책 | 문서 |
| 카테고리 페이지 | 전체 타임라인 | date |

> 이 구조는 Chirpy 테마 수정 없이 유지 가능하며, 관리 비용이 낮고, 필요 시 확장 여지를 남긴 설계다.

---

## 확장 가능성에 대한 정리

현 시점의 요구사항을 기준으로 한 선택이기 때문에, 향후 콘텐츠 규모가 커질 경우 아래 항목처럼 확장이 가능하다.

  - 사용자용 Index 페이지 도입
  - HTML 기반 커스텀 목록 구현
  - 테마 포크를 통한 카테고리 정렬 변경
