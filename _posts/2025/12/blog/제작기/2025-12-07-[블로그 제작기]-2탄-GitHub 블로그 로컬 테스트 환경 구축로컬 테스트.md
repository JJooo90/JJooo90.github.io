## **작성 목적**

GitHub Pages 기반 블로그(Jekyll 테마)를 사용할 때, 로컬 환경에서 미리 테스트하면 빌드 오류를 빠르게 확인하고 안정적으로 게시할 수 있다.  
따라서, **Ruby 설치부터 Jekyll 서버 실행까지의 전체 플로우**를 정리를 목표로 작성하였다.

---

## **1. 프로젝트 개요**

GitHub로 블로그를 구축하는 과정에서, 원활한 개발 환경을 위해 로컬에서 Jekyll 서버를 실행할 수 있는 환경을 준비한다.
즉, 사전 테스트를 위한 단계라 생각하면 된다.

> 아래 단계는 **Windows 기준**이며, macOS에서도 유사하게 적용됩니다.

---

## **2. Ruby 설치**

Jekyll은 Ruby 기반으로 동작하기 때문에 Ruby 설치가 가장 먼저 필요함.

1. RubyInstaller 다운로드  
    👉 [https://rubyinstaller.org/downloads/](https://rubyinstaller.org/downloads/)
2. **Ruby + Devkit 버전** 설치  

설치 후 버전 확인:

    ruby -v
    gem -v

---

## **3. Bundler 설치**

Bundler는 Gem 의존성을 관리하는 패키지 매니저로, 아래 명령어로 설치

    gem install bundler

설치 확인

    bundler -v

---

## **4. Jekyll 및 관련 의존성 설치**

블로그 프로젝트 폴더로 이동한 후, 필요한 의존성 설치

    cd 프로젝트_폴더
    bundle install

> 💡 **Gemfile.lock 충돌 시**  
> GitHub Pages에서 지원하는 Ruby 버전과 로컬 Ruby 버전 차이 때문에 에러가 날 수 있음 
> 이 경우 `bundle update` 혹은 `Ruby 버전` 확인

---

## **5. 로컬 서버 실행**

설치가 완료되면 Jekyll 서버를 실행

    bundle exec jekyll serve

성공적으로 실행되면 아래와 같은 메시지를 확인 할 수 있음

    Server running...
    => http://127.0.0.1:4000

브라우저에서 접속하면 블로그가 로컬 환경에서 정상적으로 로드 됨

---

## **6. 로컬 테스트 확인**

테스트 순서

 1. 페이지 이동 시 에러 없는지  
 2. 이미지 경로가 올바르게 표시되는지  
 3. 포스트 작성 후 Markdown 렌더링이 정상인지  
 4. 스타일(SCSS) 수정이 적용되는지  
 5. Chirpy, Minimal Mistakes 등 테마 기능이 제대로 작동하는지  

로컬에서 충분히 테스트한 뒤 GitHub에 push하면,  
예상하지 못한 빌드 오류를 미리 예방할 수 있다.

---

## **마무리**

GitHub Pages는 편리하지만, 로컬에서 확인하지 않고 바로 배포하면  
**Liquid 오류, 빌드 실패, SCSS 컴파일 에러** 등이 발생해 디버깅이 어려울 수 있다.
