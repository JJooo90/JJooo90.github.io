## 문제 상황

GitHub Pages + Chirpy 테마에 Giscus 댓글 기능을 설치했다.

- `_config.yml` 설정 완료
- 각 게시글에 `comments: true` 추가
- GitHub Actions 배포 성공
- DevTools에서 `giscus.app/client.js` 로드 확인

그럼에도 불구하고 **댓글이 안 보였다**.

그래서 다음과 같은 의심을 했다.

- URL 구조(`/posts/...`) 문제인가?
- permalink 설정이 잘못된 건가?
- Giscus mapping(`pathname`)이 꼬인 건가?

결론적으로 **전부 아니었다**.

---

## 실제 원인

**댓글은 정상적으로 렌더링되고 있었다.**

다만, Chirpy 테마의 기본 레이아웃 구조 때문에  
댓글 영역이 화면 아래쪽으로 밀려 있었을 뿐이었다.

Chirpy의 기본 렌더링 순서는 다음과 같다.

  - 본문 내용
  - 관련 글(Related Posts)
  - 댓글(Comments, Giscus)

문제는 **관련 글 영역이 길어질 경우**였다.

- 시리즈 글
- 관련 글이 많은 글
- Index 성격의 글

이런 경우, 관련 글 리스트가 매우 길어지고  
댓글은 그 아래에 위치하게 된다.

결과적으로:

> 스크롤을 충분히 내리지 않으면  
> 댓글이 없는 것처럼 보이는 상태가 된다.

---

## 확인 과정

DevTools에서 다음을 확인했다.

  - `giscus.app/client.js` 스크립트 존재
  - Console 메시지:
    - `[giscus] Discussion not found. A new discussion will be created if a comment/reaction is submitted.`

이는 오류가 아니라 **정상 동작 로그**였다.

- 해당 페이지에 아직 Discussion이 없다는 의미
- 첫 댓글 작성 시 자동으로 Discussion 생성

즉, Giscus 자체는 이미 정상 작동 중이었다.

---

## 오해했던 포인트 정리

### URL 문제라고 생각했던 이유

- 실제 접속 URL:
  `/posts/블로그-제작기-7탄-.../`
- 예상했던 URL:
  `/posts/2025/12/blog/제작기/.../`

하지만 이는 **permalink 설정 차이**일 뿐,
댓글 표시 여부와는 직접적인 관련이 없었다.

URL이 달라도:

  - giscus 스크립트가 삽입되면
  - 댓글은 정상적으로 렌더링된다

이번 문제의 핵심은 **URL이 아니라 레이아웃 순서**였다.

---

## 해결 방향 선택지

이 문제를 해결하는 방법은 크게 세 가지다.

### 1. 관련 글 영역 비활성화

댓글을 더 강조하고 싶다면 가장 단순한 방법이다.

  _config.yml

    related_posts: false

댓글이 본문 바로 아래에 노출된다.

---

### 2. 댓글 → 관련 글 순서로 변경

UX 관점에서 가장 자연스러운 구조다.

  - 독자가 글을 읽고
  - 바로 댓글을 확인/작성
  - 이후 관련 글 탐색

단, `_layouts/post.html` 수정이 필요하며  
테마 업데이트 시 재적용이 필요하다.

---

### 3. 관련 글을 접기 UI로 변경

관련 글은 유지하되, 댓글 접근성을 높이는 방식이다.

  - 기본 상태: 댓글 바로 노출
  - 필요 시 관련 글 펼쳐보기

시리즈 글이 많은 블로그에 적합하다.

---

## 최종 정리

이번 이슈의 본질은 다음 한 줄로 정리된다.

> **댓글이 안 보였던 게 아니라,  
> 관련 글 영역 아래에 가려져 있었던 것이다.**

Giscus 설정, URL, mapping, 권한 문제는 없었고  
순수하게 **Chirpy 레이아웃 UX 문제**였다.

이 경험을 통해 얻은 교훈은 하나다.

> 기능이 안 된다고 의심하기 전에  
> 실제로 DOM에 렌더링되고 있는지부터 확인하자.
