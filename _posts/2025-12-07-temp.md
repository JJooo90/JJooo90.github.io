# **[Blog] GitHub 블로그 로컬 테스트 환경 구축 가이드**

GitHub Pages 기반 블로그(Jekyll 테마)를 사용할 때, 로컬 환경에서 미리 테스트하면 빌드 오류를 빠르게 확인하고 안정적으로 게시할 수 있습니다.  
이번 글에서는 **Ruby 설치부터 Jekyll 서버 실행까지의 전체 플로우**를 정리합니다.

---

## **1. 프로젝트 개요**

GitHub로 블로그를 구축하는 과정에서, 원활한 개발 환경을 위해 로컬에서 Jekyll 서버를 실행할 수 있는 환경을 준비합니다.  
아래 단계는 **Windows 기준**이며, macOS에서도 유사하게 적용됩니다.

---

## **2. Ruby 설치**

Jekyll은 Ruby 기반으로 동작하기 때문에 Ruby 설치가 가장 먼저 필요합니다.

1. RubyInstaller 다운로드  
    👉 https://rubyinstaller.org/downloads/  
2. **Ruby + Devkit 버전** 설치  
3. 설치 과정 마지막에 표시되는 `ridk install` 창에서 **1, 2, 3 모두 설치**

설치 후 버전 확인:

    ruby -v
    gem -v

---

## **3. Bundler 설치**

Bundler는 Gem 의존성을 관리하는 패키지 매니저입니다.

아래 명령어로 설치합니다:

    gem install bundler

설치 확인:

    bundler -v

---

## **4. Jekyll 및 관련 의존성 설치**

블로그 프로젝트 폴더로 이동한 후 필요한 의존성을 설치합니다.

    cd 프로젝트_폴더
    bundle install

> 💡 **Gemfile.lock 충돌 시**  
> GitHub Pages에서 지원하는 Ruby 버전과 로컬 Ruby 버전 차이 때문에 에러가 날 수 있습니다.  
> 이 경우 `bundle update` 혹은 Ruby 버전 맞추기를 고려합니다.

---

## **5. 로컬 서버 실행**

설치가 완료되면 Jekyll 서버를 실행합니다.

    bundle exec jekyll serve

또는 테마에 따라:

    bundle exec jekyll serve --livereload

성공적으로 실행되면 아래와 같은 메시지가 나타납니다:

    Server running...
    => http://127.0.0.1:4000

브라우저에서 접속하면 블로그가 로컬 환경에서 정상적으로 표시됩니다.

---

## **6. 로컬 테스트 확인**

아래를 점검해보세요:

- 페이지 이동 시 에러 없는지  
- 이미지 경로가 올바르게 표시되는지  
- 포스트 작성 후 Markdown 렌더링이 정상인지  
- 스타일(SCSS) 수정이 적용되는지  
- Chirpy, Minimal Mistakes 등 테마 기능이 제대로 작동하는지  

로컬에서 충분히 테스트한 뒤 GitHub에 push하면,  
예상하지 못한 빌드 오류를 미리 예방할 수 있습니다.

---
