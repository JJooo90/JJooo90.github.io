## 1. GitHub 프로젝트를 작성하게 된 계기

개발자라면 기술 블로그는 가지고 있어야 한다는 생각을 대학 시절부터 갖고 있었다.  
tistory를 이용해서 작성하기도 해봤지만 꾸준히 작성하지는 못하고 어느 순간부터 작성을 안 했었다.

다시 해보려고 마음먹고 시작하려니, tistory 같은 블로그 서비스를 사용하기에는 뭔가 아쉽다는 생각이 들었다.  
그래도 개발자인데 GitHub를 이용해 블로그 운영을 해봐야겠다는 생각이 들었고,  
검색을 해봐도 내 입맛에 맞게 정리된 글이 없어서 내가 직접 남겨보자는 결론에 도달했다.

---

## 2. 블로그를 생성하기 위한 사전 지식

GitHub 기반 블로그 생성 방식은 크게 3가지로 나눌 수 있다.

### 1) Jekyll 기반 GitHub Pages (가장 쉬움 – 추천)

- GitHub 공식 지원  
- 다양한 테마  
- 광고 삽입 쉬움 (**이게 가장 끌린 이유**)  
- 개발자 블로그 스타일로 Markdown 작성에 최적

### 2) Static Site Generator 기반 (Hugo, Astro, Next.js 등)

- 자유도 높음  
- 디자인/기능 직접 구성 가능  
- SEO 최적화 가능  
- 난이도 높음 → 실력자용

### 3) Notion → GitHub Pages 연동 (비추천)

- 간단하지만 기능적 제약 많음

### 2) 그냥 빠르게 블로그 생성하기

1. https://github.com/cotes2020/chirpy-starter 접속  
2. **Use this template** 클릭  
3. **Create a new repository** 선택  
4. 입력  
   - Repository name: 계정명.github.io  
   - Description: 자유롭게  
   - Public  
5. **Create repository** 클릭

## 4. 기본 설정 커스터마이징 (_config.yml)

- 블로그 제목  
- 설명(description)  
- 프로필 정보  
- SNS 계정 연결  
- Google Analytics(선택)  
- 카테고리/태그 설정  

모두 `_config.yml`에서 조절한다.

---

## 5. GitHub에 push → Actions가 빌드/배포

```bash
git status       # 변경 확인
git add .
git commit -m "Initial setup"
git push origin main
```

GitHub → Actions 메뉴에서 빌드 상태를 확인할 수 있다.
