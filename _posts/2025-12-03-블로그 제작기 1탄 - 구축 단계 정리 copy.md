## 1. github 프로젝트를 작성하게 된 계기

개발자라면 기술 블로그는 가지고 있어야 한다는 생각을 대학 시절부터 갖고 있었다.
tistory를 이용해서 작성하기도 해봤지만, 꾸준히 작성하지는 못하고 어느순간부터 작성을 안 했었다.

다시 해보려고 마음 먹고 시작하려니, tistory와 같은 블로그 서비스를 하기에는 뭔가 아쉽다는 생각이 들어,  
그래도 개발자인데, github을 이용해서 블로그 운영을 해봐야겠다는 생각이 들었다.

그런데, 생각보다 복잡하고, 서칭을 통해 블로깅 방법에 대해 자세히 설명하는 글들이 내 마음에 와닫지 않았다.
그래서, 내가 제대로 작성해서, 나같은 사람이 쉽게 블로그를 할수 있는 지침이 되도록 작성해야겠다는 마음이 들었다.

결론적으로, 내가 작업해서 참고용으로 잘 작업할수 있도록 만들어 볼까 한다.

## 2. 블로그를 생성하기 위한 사전 지식

우선, github을 이용해서 하는 방법은 여러가지가 있는데, 크게 3가지로 분류할 수 있다.

1. 블로그 분류
    1. Jekyll 기반 GitHub Pages (가장 쉬움 – 추천)
        - Github에서 공식적으로 지원
        - 테마 다양
        - 광고 삽입 쉬움(**이게 가장 끌린 이유**)
        - 개발자 블로그 스타일이라 코딩 위주 글 쓰기 좋음 (작성 방법이 기본적으로 markdown임)
    2. Static Site Generator 기반 (Hugo, Astro, Next.js 등)
        - 자유도가 높음(찐 고수들을 위한 영역이랄까?)
        - 디자인/기능 마음대로 구성가능(Front 자신 있으면 요걸로 ㄱㄱ 하세요.)
        - SEO 최적화 가능(SEO(Search Engine Optimization) : 검색 엔진 최적화 - 결국 고수용)
    3. Notion → GitHub Pages 연동 (비추천)
        - 간단하나, 결국 서비스 방법이기 때문에 비추

2. 블로그 생성 로드맵 (30분이면 뚝딱 할 수 있음)
    1. GitHub 저장소 생성
    2. Jekyll 테마 선택
    3. 테마 설치
    4. Github Pages 설정
