## 1. 목적
본 문서는 블로그 콘텐츠의 3가지 항목을 명확히 정의하여, 일관된 운영과 유지보수를 가능하게 하는 것을 목적으로 한다.
- 시리즈 관리 방식  
- 카테고리 역할  
- 정렬 기준

## 2. Series 정책
### 2.1 series 필드
- `series`는 게시글 간 연관 관계를 나타내는 용도로 사용한다.
- 시리즈는 자동 정렬 기능을 제공하지 않는다.

### 2.2 series_order 필드
- 시리즈 내부의 권장 읽기 순서는 `series_order`로 관리한다.
- 숫자가 작을수록 앞선 글이다.
- 기본 단위는 10을 사용한다.

예시:

    ---
    series: "블로그 제작기"
    series_order: 30
    ---
---

### 2.3 시리즈 목록 출력 표준 (권장 구현)

- 시리즈에 속한 게시글의 상단에는, 동일한 `series` 값을 가진 게시글 목록을 출력한다.
- 목록 정렬 기준은 `date`가 아닌 `series_order`를 사용한다.
- 본 구현은 시리즈 내부 흐름을 사용자에게 명확히 안내하기 위한 표준 방식이다.

권장 구현 예시는 다음과 같다.

    {% assign series_posts = site.posts
    | where: "series", page.series
    | sort: "series_order" %}

    <div class="series-box">
    <div class="series-title">바로가기</div>
        <ul>
            {% for post in series_posts %}
            <li {% if post.url == page.url %}class="current"{% endif %}>
                <a href="{{ post.url }}">
                {{ forloop.index }}. {{ post.title }}
                </a>
            </li>
            {% endfor %}
        </ul>
    </div>

- 본 코드에서 `series_order`가 누락된 게시글은 의도된 시리즈 순서를 보장할 수 없으므로 허용하지 않는다.

## 3. Categories 정책
- 카테고리는 콘텐츠의 **분류 및 타임라인** 역할을 한다.
- 정렬 기준은 Chirpy 기본 동작(date 기준)을 따른다.
- 카테고리 페이지에서 시리즈 순서를 강제하지 않는다.
- 폴더 구조와 카테고리는 직접적인 연관이 없다.

예시:

    ---
    categories: [blog, 제작기]
    ---

---

## 4. Index 문서 정책
- Index 문서는 사용자 진입점이 아닌 **운영 정책 문서**로 정의한다.
- 시리즈별 정렬 기준 및 관리 규칙을 명문화한다.
- 사용자 UX를 위한 허브 역할은 부여하지 않는다.
- 필요 시 추후 사용자용 Index로 확장 가능하다.

---

## 5. 설계 원칙 요약
- date 기반 자동 정렬은 유지한다.
- 시리즈 흐름은 사람이 책임진다.
- 테마 수정은 최소화한다.
