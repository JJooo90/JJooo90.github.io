## 1. 미완성 게시글이 자주 발생하는 이유

깃허브 블로그를 운영하다 보면, 게시글을 작성하다가  
완성하지 못한 채로 업로드되는 상황이 자주 발생한다.

- 글을 쓰다가 중간에 멈추고
- 일단 커밋·푸시를 해두고
- 시간이 지나 다시 열어보면  
  **어디까지 썼는지, 왜 멈췄는지 기억이 나지 않는다**

이 문제는 의지나 성실성의 문제가 아니다.  
대부분 **“작성 상태를 관리하지 않는 구조”**에서 발생한다.

---

## 2. 핵심 원칙 정리
이 문제를 해결하기 위해 세 가지 원칙을 세웠다.

### 원칙 1 : **미완성 글은 시스템이 미완성임을 알 수 있어야 한다.**
### 원칙 2 : **글의 상태(State)를 파일 내부에 명시한다.**
### 원칙 3 : **다시 작성할 때, 생각을 복원하는 비용을 최소화한다.**

이 원칙을 기준으로 관리 방법을 정리했다.

---

## 3. 방법 A: Front Matter에 작성 상태를 명시하기

가장 먼저 적용한 방법은  
**게시글의 상태를 Front Matter에 명시하는 것**이다.

### 예시

```yaml
status: writing   # draft | writing | review | done
last_todo: "raw 블록과 details 충돌 예시 정리"
resume_hint: "Liquid 코드 하이라이트 문제 해결 후 예제 추가"
```

---

실제 사용 예시는 다음과 같다

```markdown
title: "[blog] 포스트 작성에서 Liquid 코드 작성하기"
categories: [blog, 삽질기록]
tags: [blog, github, Liquid]
toc: true

status: writing
last_todo: "raw 블록과 details 충돌 예시 정리"
resume_hint: "하이라이트 문제 해결 후 코드 예시 보완"
```

이렇게 하면 :
- 글을 열자마자 **현재 상태**가 보이고
- 어디까지 작업했는지 바로 파악할 수 있다.
