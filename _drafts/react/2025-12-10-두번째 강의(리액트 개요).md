## 1. React란?
- 사용자 인터페이스를 만들기 위한 javascript UI 라이브러리 

- 프레임 워크 vs 라이브러리
    - 프레임 워크 : 제어 권한이 프레임 워크에 있음
    - 라이브러리 : 제어 권한이 개발자에게 있음

## 3. JSX?
- A syntax extension to javascript ( 자바스크립트의 확장 문법 )
- Javascript + xml /HTML

- jsx 예시 코드 

```jsx
const element = <h1>Hello, world! </h1>;
// 자바스크립트 코드와 HTML 코드가 결합되어 있는 JSX 코드
```
- JSX의 역할
    - jsx는 내부적으로 xml/html 코드를 자바스크릅트로 변환하는 과정을 거치게 됨
    - jsx로 코드를 작성해도 최종적으로 자바스크립트 코드가 나오게 됨

- 리액트의 createElement() 함수
    - jsx코드를 자바스크립트 코드로 변환하는 역할을 하는 함수

- jsx를 사용하는 예시 코드
    컴포넌트 내부에서 자바스크립트 코드와 HTML 코드가 결합된 JSX를 사용하고 있음

```jsx

//JSX를 사용한 코드
    class Hello extends React.Cpmponent {
        render() {
            return <div>Hello {this.props.toWhat}</div>;            
        }
    }

    const root = ReactDom.createRoot(document.getElementById('root'));
    root.render(<Hello toWhat="world" />);


//JSX를 사용하지 않고 순수 자바스크립트 코드만을 사용
    class Hello extends React.Componet {
        render(){
            return React.createElement('div', null, 'Hello ${this.props.toWhat}' );
        }
    }

    const root = ReactDom.createRoot(document.getElementById('root'));
    root.render(React.createElement(Hello, {toWhat: 'World'}, null));

```

- createElement() 함수의 파라미터
    - type : element의 유형
    - props : 속성등
    - children : 현재 element가 포함하고 있는 자식 element

```bash
    React.createElement(
        type,
        [props],
        [...children]
    )
```

- jsx 사용법
XML/HTML 코드 사이에 중괄호를 사용하여 자바스크립트 코드를 묶어주면 됨
